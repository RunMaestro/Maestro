Verification & open decisions

Runtime-validated problems (current main, 85788c96)

  • Busy pill shows elapsed/tokens only; no click-through on primary label; background command counts absent from primary UI.
  • Collapsed sidebar has no creation affordance; New Agent lives in the footer, not with the list / New Group.
  • Interface Font select unmounts to “Loading fonts...” on first interaction, closing the native dropdown; single font drives UI and terminal.
  • Cmd+K “term” returns New Terminal with no shortcut hint; “Switch AI/Shell Mode” doesn't match “term”; ⌘J semantics differ per surface.
  • Wizard requires a blank agent name with no explanation, then auto-advances into token-spending playbook generation without model/scope disclosure.

Caveats & open product decisions

  • macOS-only caveat: native NSMenu rendering, accelerator display behavior, and font enumeration are unverified on this Windows host; menu visuals here are a faithful mock, gated on on-device QA.
  • O8 — ⌘J ownership: Switch AI/Shell Mode vs New Terminal is a backward-compatibility product decision; this prototype shows Option A (toggle = ⌘J, new terminal = ⌘⇧J). Invariants hold under either option.
  • O2 — Inspector placement: bottom drawer (shown) vs right panel.
  • O1/O5 — shortcut collisions: ⌘N (New Agent) and ⌘⇧J (New Terminal) need a collision audit against DEFAULT_SHORTCUTS before binding.
  • No fabricated data anywhere: no percent/ETA, no provider health, no glyph-coverage counts; quiet state is neutral measured silence; Agent-only is always the preselected path; scan is local until an explicitly disclosed send.
